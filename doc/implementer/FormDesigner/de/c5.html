5. Funktionen für den Action-Parameter

Funktionen können zu verschiedenen Zwecken eingesetzt werden. So können z.B. Maskenwerte in Abhängigkeit von anderen Werten umgeformt, berechnet, oder aus der Datenbank geholt werden, oder sie können auch Datenbanksequenzen erhöhen und andere Datenmanipulationen durchführen. Eine Funktion wird bei dem Maskenobjekt - Parameter Action - spezifiziert, in dem der Wert nach der Ausführung der Funktion erscheinen soll. In der Action Spezifikation wird ein F oder f zur Identifikation eines Funktionsaufrufes einer Funktion vorangestellt. Das große F besagt, dass die Funktion bei jedem Drücken der Tab- oder Enter-Taste (Nummernblock) ausgeführt wird, während mit klein-f angegebene Funktionen bei Klick auf den Knopf Insert oder Strg+i ausgeführt werden - also nach Eingabe eines kompletten Datensatzes.

Es gibt eine Reihe von Funktionen, die im Folgenden beschrieben werden:

fromDB
Diese Funktion holt aus der Datenbank einen Wert und zeigt ihn im zugehörigen Feld an. Abhängig vom Rückgabewert kann ein Perl-Script ausgeführt werden.

Syntax:
    F|f fromDB table column1 [column2 section]
       SELECT column1 FROM table WHERE column2=(Wert_von_Objekt_section)

oder 
    F|f fromDB table column1 column2 section [where-clause_1] [where_clause_2] ...
       SELECT column1 FROM table
       WHERE column2=(Wert_von_Objekt_section)
       AND where-clause_1 AND where_clause_2


Beispiele: 
    F fromDB codes long_name db_code sec110
    F fromDB weight sum(weight)
    F fromDB codes long_name codes.db_code sec130 codes.class='country'
existDBID
In Abhängigkeit von den Maskenwerten wird ein Wert aus der Datenbank geholt und im zugehörigen Feld angezeigt.

Syntax:
    F|f existDBID table column [section1 column1 [section2 column2] ... [sectionn columnn]]
      SELECT column FROM table
      WHERE column1=(Wert_von_Objekt_section1)
      AND column2=(Wert_von_Objekt_section2)
      ...
      AND columnn=(Wert_von_Objekt_sectionn)


Beispiele: 
    F existDBID codes db_code sec001 short_name col002 class
    F existDBID entry db_animal sec130 ext_animal sec013 db_unit
getDBID
Dieselbe Funktionalität wie existDBID, jedoch zusätzlich kann in Abhängigkeit vom Rückgabewert ein Perl-Script ausgeführt werden.
Next_seq_DBID
Diese Funktion liefert in Abhängigkeit von anderen Maskenwerten eine neue, nichtbenutzte Datenbank-Id zurück. Existiert zu der spezifizierten Bedingung ein Datensatz, wird die Datenbank-Id zurückgegeben.
In der Datenbank muss entweder eine Sequenz mit dem Namen seq_table_db_id (mit table=Tabellenname) existieren, oder in der Modeldatei muss für die entsprechende Tabelle unter %table{TABLE}{CONTRAINTS}{SEQUENCE} ein Name definiert sein.

Diese Funktion nutzt die nextval()-Funktion von PostgreSQL.

Syntax: 
    f Next_seq_DBID table column section1 column1 [section2 column2 ... [sectionn columnn]]
Beispiel: 
    f Next_seq_DBID address db_address sec001 ext_address
In das Feld mit der Bezeichnug sec001 wird eine Adresse eingegeben. Existiert die in address.ext_address wird der Wert aus address.db_address zurückgegeben. Anderenfalls wird die Sequenz %address{TABLE}{CONSTRAINTS}{SEQUENCE}[0], oder falls die nicht existiert, die Sequenz seq_address_db_id inkrementiert.
getNextDBID
Diese Funktion hat die gleiche Funkionalität wie Next_seq_DBID benutzt für die Sequenz aber eine extra Tabelle. Erforderlich ist eine Tabelle db_id mit einer Spalte max_db_id (CREATE TABLE db_id (max_db_id int) ).

Syntax: 
    f getNextDBID table column [section1 column1 [section2 column2] ... [sectionn columnn]]
Beispiel: 
    f Next_seq_DBID  db_address sec001 ext_address
sum
Berechnet die Summe zweier Felder.

Syntax: 
    F|f sum section1 section2 [FIX=Anzahl_Nachkommastellen]
Beispiel: 
    F sum sec200 sec210 FIX=2
sub
Subtraktion zweier Felder.

Syntax: 
    F|f sub section1 section2 [FIX=Anzahl_Nachkommastellen]
Beispiel: 
    F sub sec20 sec21
multi
Multiplikation zweier Felder.

Syntax: 
    F|f multi section1 section2 [FIX=Anzahl_Nachkommastellen]
Beispiel: 
    F multi sec020 sec030
Siehe auch die Demo-Maske $APIIS_HOME/test/demo/EuroCalc.form
div
Division zweier Felder.

Syntax: 
    F|f div section1 section2 [FIX=Anzahl_Nachkommastellen]
Beispiel: 
    F div col020 col030 FIX=10
Siehe auch die Demo-Maske $APIIS_HOME/test/demo/EuroCalc.form
concat
Verbindet den Inhalt mehrerer Felder zu einer einzigen Zeichenkette und liefert diese zurück. Optional kann ein Trennzeichen oder Zeichenkette angegeben werden.

Syntax: 
    F|f concat section1 section2 ... [delim=char|string]
Beispiel: 
    F concat sec200 sec210 sec220 delim=|
Siehe auch die Demo-Maske $APIIS_HOME/test/demo/concat.form
split
Unterteilt eine Zeichenkette am Trennzeichen in mehrere Teile. Die Teilzeichenketten werden in verschiedenen Feldern angezeigt. Das Trennzeichen kann auch aus mehreren Zeichen bestehen. Gibt es mehr Teilzeichenketten als spezifizierte Felder, werden nur soviel Teilketten gebildet, wie Felder für die Anzeige vorhanden sind.

Syntax: 
    F|f split destination_section1 [destination_section2 ...] delim=char|string
Beispiel: 
    F split sec120 sec130 sec140 delim=|
Siehe auch die Demo-Maske $APIIS_HOME/test/demo/split.form
fetchlist
Holt ein oder mehrere Datensätze aus der Datenbank und packt diese in ein mehrzeiliges Listenfeld (Typ M), oder in einzeilige Felder vom Type E,D,B oder N. Einzeilige Felder zeigen maximal einen Datensatz. Ein Aufruf mit klein-f ist auch möglich aber wohl nicht sinnvoll, da diese Funktion allein dem Anzeigen von Daten dient.

Syntax: 
    F|f fetchlist table column1[|column2 ...|columnn] section columnW [delimiter] [order=columnS]
       SELECT  column1,column2,columnn FROM table
       WHERE columnW=(Wert_von_Objekt_section)
       ORDER BY columnS

       delimiter: Trennzeichen oder -zeichenkette der angezeigten Datenbankspalten
                  dabei bedeutet ein b eine Leerstelle. Beispiel: b|b = Leerstelle|Leerstelle
Hinweis: Die Parameter dürfen auf keinen Fall mit Komma getrennt werden! Stattdessen ist das Zeichen '|' zu verwenden.

Beispiel: 
    F fetchlist animal name|short_name|origin|birth_dt|ai_code sec007 db_animal b:b
SQL
Ausführen von SQL-Statements ohne Überprüfen der Check-Regeln.

Syntax: 
    F|f SQL sql-statement

    Feldwerte können in sql-statement als Sektionsname - eingeschlossen in $ - benutzt werden.
Beispiele: 
    f SQL insert into entry (ext_animal|db_animal|db_unit|last_change_dt|last_change_user)
           values ($col012$| $col007$| $col005$| '$now'| '$col012c$')

    f SQL delete from transfer where db_id=$sec123$;delete from entry where db_id=$sec123$
Für Listenfelder (TYPE=M):
Selektierte Werte werden in das Listenfeld eingetragen. Eventuell vorhandene Werte werden vorher gelöscht.

Beispiel: 
    f SQL SELECT username FROM users;
exeCommand
Ausführen von Kommandos definiert in einer Section oder in GENERAL.

Syntax: 
    F|f exeCommand section label
Beispiele: 
    [secxxx]
    ACTION=f exeCommand sec4711 LABEL_1

    [sec4711]
    LABEL_1=print "sec4711:  $$form{sec4711}{DATA} \n"


    [secyyy]
    # Defaultparameter für Anrede setzen
    ACTION=F exeCommand sec0815 DEFAULT

    [sec0815]
    DEFAULT=$$form{sec0815}{DATA}='Herr'
