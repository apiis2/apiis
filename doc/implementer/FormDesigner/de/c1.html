2. Das Hauptfenster

Beim Start des FormDesigners erscheint zunächst das Hauptfenster auf dem Bildschirm (Abbildung 1). Das Fenster besteht aus mehreren Teilen: einer Menüleiste, einem mit Database bezeichneten Bereich, einem Feld zur Auswahl der Maskenobjekte und zwei Knöpfen.

Die Menüleiste besteht aus vier Menüs: File, Edit, Options und Help.

In dem mit Database bezeichneten Bereich sind die Felder Tables und Columns zunächst leer. Nach Angabe einer Model-Datei werden im Feld Tables alle definierten Tabellen angezeigt. Durch Anklicken eines Tabellenamens werden im Feld Columns die dazugehörigen Spalten aufgeführt (Abbildung 2).

Im Listenfeld Form Field Types sind alle verfügbaren Objekttypen aufgelistet und können per Mausklick ausgewählt werden. Durch Wahl eines Typs wird der darunterbefindliche rechte Knopf aktiv und bringt ein dem Typ entsprechendes Konfigurationsfenster auf den Schirm.

Der linke Knopf Update dient zum Aktualisieren der Maske. Der rechte Knopf ist zu Beginn unbeschriftet.

Die Menüs

 Dieser Abschnitt enthält eine kurze Beschreibung über die Wirkungsweise der einzelnen Menüpunkte.

File

 File->New
Neue Maske erstellen: Startet eine Dialogbox zum Konfigurieren einer neuen Maske.

File->Open...
Öffnet eine Form-Datei zum Editieren.

File->Save
Maskendefinitionen in eine Form-Datei speichern. Wenn noch kein Dateiname angegeben wurde, wird eine Datei untitled.form angelegt.

File->Save as...
Maskendefinition unter Angabe eines Dateinames in eine Form-Datei speichern.

File->Exit
Schließen aller offenen Fenster und Beenden des FormDesigners.


Edit

 Edit->GENERAL
Dialogbox zum Konfigurieren der generellen Maskenparameter.
Für jedes auf der Maske plazierte Objekt wird ein Eintrag in das Menü aufgenommen. Das Edit-Menü kann an der gestrichelten Linie abgerissen werden.


Options

 Options->Debug
Schaltet Debug-Meldungen an oder aus. Die Meldungen werden in der Shell, in der der FormDesigner gestartet wurde, ausgegeben.
Entspricht Form -d

Options->SQL-Print
On oder Off. SQL-Print On gibt das letzte SQL-Statement in der Shell aus.

Options->set APIIS_HOME...
Zum Ändern oder Setzen der Umgebungsvariable $APIIS_HOME.

Options->set APIIS_LOCAL...
Zum Ändern oder Setzen der Umgebungsvariable $APIIS_LOCAL.

Options->Browser...
Ein HTML-Browser dient zum Anzeigen der Hilfe-Seiten. Ein bevorzugter Browser kann hier gewählen werden. Um die Einstellung permanent zu machen, kann in der Datei $APIIS_HOME/apiisrc ein Eintrag folgender Art gemacht werden: 
            $browser = 'netscape';


Options->File Selector...
Der FormDesigner stellt zwei verschiedene Datei-Dialog-Boxen zur Verfügung: FBox und FileSelector. FBox ist voreingestellt, weil er gewisse Vorteile bei der Auswahl von Dateien hat, z.B. anzuzeigende Dateitypen können spezifiziert werden. Leider läuft FBox auf einigen Computern nicht korrekt, da das Einstiegsverzeichniss '/' (root-Verzeichnis) ist. Normalerweise ist das Einstiegsverzeichnis $APIIS_LOCAL b.z.w. $APIIS_LOCAL/model/forms falls dieses Verzeichnis existiert. Um die Einstellung permanent auf FileSelector zu setzen, kann in der Datei $APIIS_HOME/apiisrc ein Eintrag folgender Art gemacht werden: 
            $fileselector = 'FileSelector';


Der Patch von Slaven Rezic Patch::SREZIC::Tk::FBox behebt das Problem mit FBox.


Help

 Help->Contents
Mit diesem Menüeintrag wird die Dokumentation von FormDesigner in einem HTML-Browser geöffnet.

Help->About Fonts
Beschreibt die Unterschiede zwischen MS Windows- und X11-Zeichensätzen und wie systemunabhängige Zeichensätze erstellt werden können.

Help->About...
Zeigt Informationen zum FormDesigner an, z.B. die Versionsnummer.
